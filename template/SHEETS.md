# SHEETS.md — Спецификация шаблонных листов SERPlux

**Версия:** 1.0  
**Дата:** 2026-07-12  
**Создатель:** initTemplateSheets() в apps_script.gs  

---

## Обзор

Каждая таблица клиента SERPlux состоит из **6 листов** (БЕЗ Лист1):

| Лист | Назначение | Заполнение | Редактор |
|------|-----------|-----------|----------|
| **Настройки** | Параметры сбора | Вручную пользователем | Пользователь |
| **Данные** | Кэш выдачи (плоский формат) | Автоматически прогоном | Система |
| **Отчёт** | Матрица позиций по гео | Автоматически прогоном | Система |
| **Эталон разметки** | Кэш тональности по доменам | Автоматически нейронкой | Система / Пользователь |
| **Спорные** | Накопитель для deep-режима (v2) | Автоматически при сомнении | Система |
| **Лог** | История событий | Автоматически прогоном | Система |

---

## Лист 1: «Настройки»

### Назначение
Параметры сбора и настройки клиента. Таблица ключ–значение.

### Структура

| Колонка A | Колонка B | Колонка C |
|-----------|----------|----------|
| **Ключ** | **Значение** | **Подсказка** |
| client_id | (пусто) | ID клиента (выпадающий список) |
| depth | 10 | Глубина сбора: 10, 20, 50 или 100 |
| with_labels | true | Включить разметку: true или false |
| label_mode | auto | Режим разметки: auto или deep |
| date | today | Дата сбора: today или YYYY-MM-DD |
| force_relabel | false | Принудительная переразметка: true или false |
| force_rebuild_report | false | Перестроить отчёт: true или false |
| report_date | latest | Дата отчёта: latest или YYYY-MM-DD |
| provider_chain | opencode-zen | Цепочка провайдеров LLM |
| status | idle | Статус последнего прогона (обновляется автоматически) |

### Примечания
- **Формулировка:** инициализируется функцией `initSettingsSheetSafe()` (вызывается автоматически или вручную)
- **Data Validation:** client_id имеет выпадающий список из API `/clients`
- **Читается:** webhook при каждом запросе `/run`
- **Обновляется:** только Apps Script (для status), остальное — ручное редактирование

## Лист 2: «Данные»

### Назначение
Плоский кэш выдачи для фильтрации и анализа. Полная перезапись при каждом прогоне.

### Структура

| Дата | Поисковая система | Субъект/Запрос | Гео | Позиция | URL | Домен | Сниппет | Метка |
|------|------------------|----------------|-----|---------|-----|-------|---------|-------|

### Типы данных

| Колонка | Тип | Пример | Примечание |
|---------|-----|--------|-----------|
| Дата | YYYY-MM-DD | 2026-07-12 | Дата сбора |
| Поисковая система | Строка | Google, Яндекс, Яндекс.com | Читаемое имя |
| Субъект/Запрос | Строка | juri sudheimer | Из профиля клиента |
| Гео | Строка | Lithuania | Реальная страна |
| Позиция | INT | 1, 2, 3… | 1-based |
| URL | URL | https://example.com/news | Полный URL |
| Домен | Строка | example.com | Извлечено из URL |
| Сниппет | Строка (до 500 символов) | Текст сниппета… | Из Topvisor |
| Метка | Enum | positive, negative, neutral | По тональности |

### Примечания
- **Создаётся:** функцией `_ensureDataSheet()` с заголовками
- **Заполняется:** exporter.py при каждом прогоне
- **Очищается полностью:** перед новой записью (`worksheet.clear()`)
- **Читается:** для фильтрации, анализа, экспорта в BI-системы
- **Без Лист1:** структура исключена из шаблона (см. CANON.md)

## Лист 3: «Отчёт»

### Назначение
Матрица позиций по субъектам × гео. Накопительный режим (новые версии сверху).

### Структура (пример для 2 субъектов, 2 гео, depth=10)

```
Позиции Google на 12.7.2026

                      juri sudheimer                  |              john doe
Lithuania | URL       Germany | URL                    | Lithuania | URL       Germany | URL
1         | link1      5      | link5                  | 3         | link3      10     | link10
2         | link2      6      | link6                  | 4         | link4      11     | link11
...       | ...        ...    | ...                    | ...       | ...        ...    | ...
```

### Раскладка колонок (0-indexed)

**Вертикальное расположение:**
- Строка 1: `Позиции {searcher} на {DD.MM.YYYY}` (заголовок версии)
- Строка 2: Пустой буфер
- Строка 3: Имена субъектов (крупный шрифт, в правых колонках блоков)
- Строка 4: Гео-подзаголовки (в левых колонках блоков)
- Строки 5–N: Номера позиций + URL (по depth)

**Горизонтальное распределение субъектов:**
- Субъект 1: колонки B (гео+позиции), C (имя+URL)
- Буфер 3 колонки: D, E, F
- Субъект 2: колонки G (гео+позиции), H (имя+URL)
- Буфер 1 колонка: I
- Субъект 3: колонки J (гео+позиции), K (имя+URL)
- Буфер 1 колонка: L
- Субъект N: колонки M+ (по формуле)

### Цветовая разметка

Заливка применяется **только к ячейке номера позиции** (левая колонка):

| Цвет | Sentiment | Значение |
|------|-----------|----------|
| 🟢 Зелёный (#00FF00 or light green) | positive | Позитивная тональность |
| 🔴 Красный (#FF0000 or light red) | negative | Негативная тональность |
| 🟡 Жёлтый (#FFFF00 or light yellow) | neutral | Нейтральная тональность |

### Накопительный режим

- **Максимум версий:** 10
- **Новая версия:** вставляется СВЕРХУ через `insertDimension` (сдвиг вниз старых)
- **Удаление:** когда > 10 версий, самые старые (снизу) обрезаются через `deleteDimension`
- **Признак начала версии:** строка начинается с `"Позиции "` и содержит `" на "`

### Примечания
- **Создаётся:** функцией `_ensureReportSheet()` пустой
- **Заполняется:** reporter.py при прогоне (функция `build_report`)
- **Размер:** динамический, зависит от количества гео и субъектов в профиле клиента
- **Буферы:** отделяют блоки субъектов, не содержат данные
- **Лист1 исключён:** каноническая раскладка снята в CANON.md

## Лист 4: «Эталон разметки»

### Назначение
Кэш тональности по парам (domain, query, geo). Пополняется автоматически при прогонах.

### Структура

| domain | query | geo | sentiment | source |
|--------|-------|-----|-----------|--------|

### Типы данных

| Колонка | Тип | Пример | Примечание |
|---------|-----|--------|-----------|
| domain | Строка | example.com | Домен из URL |
| query | Строка (lowercase) | juri sudheimer | Имя субъекта, НЕ страна |
| geo | Строка | Lithuania | Реальная страна, НЕ константа |
| sentiment | Enum | positive, negative, neutral | По LLM или вручную |
| source | Enum | auto, manual_l1 | Источник разметки |

### Примечания
- **Создаётся:** функцией `_ensureEtalonSheet()` с заголовками
- **Первичный ключ:** (domain, query, geo)
- **Заполняется:** labeler.py при прогоне (сохраняет найденные метки)
- **Приоритет:** ручная разметка (source=manual_l1) не перезаписывается автоматикой
- **Читается:** на начало следующего прогона (кэш для ускорения)
- **Редактируется:** вручную если нужно исправить разметку

## Лист 5: «Спорные»

### Назначение
Накопитель для спорных/неуверенных разметок. Задел под deep-режим (v2.0).

### Структура

| domain | query | geo | url | причина |
|--------|-------|-----|-----|---------|

### Типы данных

| Колонка | Тип | Пример | Примечание |
|---------|-----|--------|-----------|
| domain | Строка | disputed.com | Домен из URL |
| query | Строка | juri sudheimer | Имя субъекта |
| geo | Строка | Lithuania | Реальная страна |
| url | URL | https://disputed.com/news/123 | Полный URL |
| причина | Строка | double_meaning, low_confidence | Описание |

### Примечания
- **Создаётся:** функцией `_ensureDisputedSheet()` с заголовками
- **Заполняется:** labeler.py при low_confidence или явном спорном тексте
- **Пока:** используется только как логирование (накопитель)
- **В v2.0:** deep-режим будет читать эту таблицу, заходить на страницы, доразмечать
- **Не удаляется:** только пополняется (для истории спорных случаев)

## Лист 6: «Лог»

### Назначение
История событий (запуски, ошибки, статусы).

### Структура

| Дата | Время | Событие | Статус | Детали |
|------|-------|--------|--------|--------|

### Примеры событий

- `2026-07-12 14:35` | `Запуск сбора` | `OK` | Собрано 150 позиций
- `2026-07-12 14:40` | `Разметка` | `OK` | Размечено 140 (10 из кэша)
- `2026-07-12 14:45` | `Экспорт` | `OK` | Выгружено 150 строк в Данные
- `2026-07-12 14:50` | `Построение отчёта` | `OK` | Добавлена новая версия

### Примечания
- **Создаётся:** функцией `_ensureLogSheet()` пустой
- **Заполняется:** webhook.py при каждом прогоне (статусные сообщения)
- **Читается:** для отладки и аудита
- **Ротирует:** старые записи остаются (нет лимита)

## Как пересоздать шаблон с нуля

Если листы повредились или нужно сбросить:

1. **В редакторе Apps Script:**
   ```javascript
   initTemplateSheets();  // Запустить вручную (Run)
   ```

2. **В таблице:**
   - SERPlux → ⚙ Настройки → [+] Инициализировать настройки
   - SERPlux → ⚙ Настройки → [⟳] Пересоздать лист Настройки

3. **Результат:** 6 листов с заголовками, данные сохраняются

- **Лист1:** исключён (матричная структура снята в CANON.md)
- **Внешние файлы:** regions_map.json, credentials.json НЕ в таблице
- **Мастер-таблица:** текущая рабочая таблица, источник шаблона

---

## Миграция

При обновлении SERPlux (новые версии листов):
1. Запустите `initTemplateSheets()` — добавит недостающие листы
2. Существующие листы не дублируются (идемпотентна)
3. Данные в старых листах сохраняются
