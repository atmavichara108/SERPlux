# topvisor API — выжимка для SERPlux

База API v2. Аутентификация: заголовки User-Id и Authorization (Bearer API-key).
Ключи из env: TOPVISOR_USER_ID, TOPVISOR_API_KEY. НИКОГДА не хардкодить.
Эндпоинт: POST https://api.topvisor.com/v2/json/<service>/<method>
Официальный OpenAPI: https://github.com/topvisor/topvisor-openapi/

## Механика сбора выдачи — ДВУХШАГОВАЯ, асинхронная

Снимок выдачи (ТОП всех сайтов) собирается так:

ШАГ 1. Запустить проверку со снимком:
  метод: edit/positions_2/checker/go
  параметры:
    filters: [фильтр по project_id]
    regions_indexes: [список region_index]
    do_snapshots: 1   # 1 = собирать снимок (Яндекс ТОП-50, Google = глубина проекта)
  возвращает: { result: { projectsIds: [...] } }
  ВАЖНО: глубина Google-снимка = глубине сбора позиций проекта.
  Для ТОП-100 убедиться, что проект настроен на нужную глубину
  (проверить в кабинете или выставить через API настроек проекта).

ШАГ 2. Опросить готовность (поллинг):
  метод: get/positions_2/history с fields ["name","positions_percent"]
  или статус проверки. Ждать пока процент = 100. Пауза 10 сек, таймаут 600 сек.

ШАГ 3. Забрать снимок:
  сервис: snapshots_2
  метод: get/snapshots_2/history
  параметры: project_id, regions_indexes, dates (дата сегодняшнего сбора)
  возвращает: историю снимков выдачи — список URL сайтов с позициями и сниппетами.

## Гео = region_index

region_index — это ключ региона в проекте (не страна напрямую, а регион/город ПС).
Узнать индексы: get/projects_2/projects с show_searchers_and_regions=1 -> поле index.
Найти region key по названию: get/system_2/common/regions
  параметры: searcher_key (1=Яндекс и т.д.), search ("Москва").
Список гео из ТЗ заранее сопоставить с region_index и записать в лист Настройки.

## Поисковые системы (searcher_key)

В проекте topvisor каждая ПС+регион добавлены отдельно. Маппинг searcher из
нашего Row на searcher проекта определить по настройкам скопированного проекта:
  google      -> Google
  yandex_ru   -> Яндекс (ru)
  yandex_com  -> Яндекс (com)

## Глубина

do_snapshots в edit/positions_2/checker/go: 0=нет снимка, 1=снимок.
Реальная глубина Google-снимка определяется глубиной сбора позиций проекта.
Параметр depth из нашего config -> при необходимости менять настройку проекта
перед запуском (метод правки проекта). На старте: проверить текущую глубину
проекта вручную в кабинете, задокументировать ограничение в README.

## Лимиты и стоимость

Проверки тарифицируются по балансу аккаунта. Каждый запуск проверки = расход.
do_snapshots увеличивает стоимость проверки. Логировать число запусков.
При масштабировании (много связок) — следить за балансом, обрабатывать
ошибку нехватки средств как частичный сбой (логировать, не падать).

## Точные имена полей ответа

Точные ключи JSON-ответа get/snapshots_2/history уточнить из OpenAPI-репозитория
topvisor на GitHub во время реализации (webfetch). Не угадывать структуру —
свериться с реальной схемой.
