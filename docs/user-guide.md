# SERPlux — Руководство пользователя

**Версия:** 1.0  
**Дата:** 2026-07-11  
**Статус:** Актуально

---

## Что это и что вы получаете

SERPlux автоматически собирает поисковую выдачу Google и Яндекс по вашим запросам и гео, размечает тональность каждого URL (позитив/негатив/нейтрал) и выгружает результат в Google Sheets.

**Вы получаете:**
- Таблицу с позициями ваших субъектов (персон, компаний) в поисковой выдаче
- Цветовую разметку тональности (зелёный = позитив, красный = негатив, жёлтый = нейтрал)
- Матрицу-отчёт для визуального анализа по гео и субъектам
- Историю изменений (последние 10 версий отчёта хранятся автоматически)

**Интерфейс:** Google Sheets (меню SERPlux + лист «Настройки»). Веб-интерфейс не используется.

## Как запустить

### Первый запуск (настройка)

1. **Откройте таблицу** — меню SERPlux появится автоматически.

2. **Инициализируйте настройки:**  
   SERPlux → Настройки → [+] Инициализировать настройки

3. **Установите URL сервера:**  
   SERPlux → Настройки → Установить URL сервера  
   Введите: `https://<ваш-домен>`

4. **Установите секрет:**  
   SERPlux → Настройки → Установить секрет  
   Введите `WEBHOOK_SECRET` из `.env` на сервере.

5. **Выберите клиента:**  
   На листе «Настройки» выберите `client_id` из выпадающего списка.

### Добавление нового клиента

Подробная инструкция: [docs/onboarding-client.md](onboarding-client.md)

**Кратко:**
1. Создайте копию этой таблицы в Google Drive
2. SERPlux → ⚙ Настройки → [+] Инициализировать настройки → запустите `initTemplateSheets()`
3. SERPlux → [К] Клиенты → Добавить клиента → укажите client_id, имя, project_id, гео, поисковики
4. Добавьте запросы (субъектов) в профиль клиента
5. SERPlux → ▶ Запустить сбор

### Запуск сбора

1. **SERPlux → ▶ Запустить сбор**

2. **Диалог подтверждения:** показывает параметры (клиент, глубина, разметка, дата). Нажмите «Да».

3. **Проверка статуса:**  
   SERPlux →  Проверить статус  
   Показывает: `running` (выполняется), `ok` (завершён), `error` (ошибка).

4. **Результат:**
   - Лист «Данные» — плоский формат с заголовками (Дата, ПС, Субъект, Гео, Позиция, URL, Домен, Сниппет, Метка)
   - Лист «Отчёт» — матрица с цветовой индикацией (накопительный режим, новые версии сверху)
   - Лист «Лог» — запись о запуске

### Разметка без повторного сбора

Если данные уже собраны, но нужно разметить их (например, другим режимом):

1. На листе «Настройки» выберите режим `label_mode` и `force_relabel`.
2. **SERPlux → Разметить собранные данные**

### Построение отчёта за дату

1. **SERPlux → [>] Построить отчёт за дату...**
2. Выберите дату из списка или оставьте пустым для последней.
3. Отчёт построен на листе «Отчёт».

## Где смотреть результат

### Лист «Данные» (кэш выдачи)

Плоский формат для фильтрации и анализа.

**Заголовки:**
```
Дата | Поисковая система | Субъект/Запрос | Гео | Позиция | URL | Домен | Сниппет | Метка
```

**Пример:**
```
2026-07-11 | Google | juri sudheimer | Lithuania | 1 | https://example.com | example.com | Сниппет... | positive
```

### Лист «Отчёт» (матрица)

Матрица для визуального анализа позиций. **Размер матрицы зависит от количества гео и субъектов, выбранных в профиле клиента.**

**Как читать:**
- **Строка 1:** заголовок версии (`Позиции Google на 11.7.2026`)
- **Строка 3:** имена субъектов (крупным шрифтом, в правых колонках блоков) — столько, сколько добавлено в профиль
- **Строка 4:** гео-подзаголовки (Lithuania, Germany, United Kingdom…) — столько, сколько выбрано в профиле клиента
- **Строки 5–N:** номера позиций (в левых колонках, с цветовой заливкой) + URL (в правых колонках). N зависит от `depth` (обычно 10)
- **Буферные колонки:** пустые, без данных и цвета

**Цветовая индикация:**
-  Зелёный — positive (позитивная тональность)
- 🔴 Красный — negative (негативная тональность)
- 🟡 Жёлтый — neutral (нейтральная тональность)

**Накопительный режим:** каждый новый прогон вставляет блок СВЕРХУ, старые версии сдвигаются вниз. Хранится 10 версий.

**Раскладка колонок (для 4 субъектов):**
- Субъект 1: B (гео+номера+цвет), C (имя+URL)
- Буфер: D, E, F (пустые)
- Субъект 2: G (гео+номера+цвет), H (имя+URL)
- Буфер: I (пустая)
- Субъект 3: J (гео+номера+цвет), K (имя+URL)
- Буфер: L (пустая)
- Субъект 4: M (гео+номера+цвет), N (имя+URL)

## Настройки

### Лист «Настройки»

Формат: таблица ключ–значение (колонка A = ключ, B = значение, C = подсказка).

| Ключ | Значение по умолчанию | Описание |
|------|----------------------|----------|
| `client_id` | (пусто) | ID клиента (выпадающий список) |
| `depth` | `10` | Глубина сбора: 10, 20, 50 или 100 |
| `with_labels` | `true` | Включить разметку: `true` или `false` |
| `label_mode` | `auto` | Режим разметки: `auto` (дефолт) или `deep` (v2) |
| `date` | `today` | Дата сбора: `today` или `YYYY-MM-DD` |
| `force_relabel` | `false` | Принудительная переразметка: `true` или `false` |
| `force_rebuild_report` | `false` | Перестроить отчёт: `true` или `false` |
| `report_date` | `latest` | Дата отчёта: `latest` или `YYYY-MM-DD` |
| `provider_chain` | `opencode-zen` | Цепочка провайдеров LLM |
| `status` | `idle` | Статус последнего прогона (обновляется автоматически) |

### Режимы разметки

- **`auto` (дефолт):** справочник доменов → LLM по сниппету → neutral при ошибке. Нулевая стоимость для известных доменов.
- **`deep`:** заглушка для v2 (уточнение neutral по контенту страницы).

## Эталон разметки и Спорные

### Что это

В вашей таблице есть два специальных листа:

- **«Эталон разметки»** — кэш разметки по доменам. Пополняется автоматически при каждом прогоне (нейронка сохраняет найденные тональности). Это справочник, чтобы не переразмечать один и тот же домен дважды.
  
- **«Спорные»** — накопитель для будущих улучшений (v2.0). Если нейронка сомневается, URL попадает сюда для ручной проверки или доразметки по контенту страницы.

### Структура листов

**Лист «Эталон разметки»:**
```
domain          | query           | geo        | sentiment | source
example.com     | juri sudheimer  | Lithuania  | positive  | auto
competitor.com  | juri sudheimer  | Lithuania  | negative  | auto
news.com        | john doe        | Germany    | neutral   | auto
```

**Лист «Спорные»:**
```
domain          | query           | geo        | url                           | причина
disputed.com    | juri sudheimer  | Lithuania  | https://disputed.com/news/123 | double_meaning
```

### Как использовать

**Автоматический режим (по умолчанию):**
- При каждом `SERPlux → ▶ Запустить сбор` нейронка:
  1. Проверяет кэш (лист «Эталон разметки») — если домен там, берёт готовую метку
  2. Если домена нет — размечает по сниппету и сохраняет в кэш
  3. Спорные домены накапливаются в листе «Спорные»

**Ручная правка (если нужно):**
- Откройте лист «Эталон разметки» и отредактируйте sentiment вручную
- При следующем запуске эта разметка будет использована (кэш имеет приоритет)

**Будущий режим (v2.0):**
- Deep-режим будет читать лист «Спорные» и заходить на страницы для полной разметки
- Подробнее: docs/roadmap-2.0.md

# Технические детали

## Архитектура

```
topvisor.py   — сбор снимков через topvisor Snapshots API
collector.py  — оркестрация по всем связкам searcher × geo
labeler.py    — разметка тональности (DeepSeek/Zen, кэш в SQLite)
storage.py    — SQLite: хранение результатов и кэш меток
exporter.py   — выгрузка кэша на лист «Данные» Google Sheets
reporter.py   — построение матрицы-отчёта на лист «Отчёт» (накопительный режим)
webhook.py    — FastAPI сервис, принимает запросы из Sheets
main.py       — точка входа, полный пайплайн
```

## API webhook

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/health` | Health-check (без авторизации) |
| `GET` | `/status` | Статус последнего прогона |
| `POST` | `/run` | Запустить пайплайн |
| `GET` | `/clients` | Список клиентов |
| `POST` | `/clients` | Создать клиента |
| `GET` | `/clients/{client_id}` | Профиль клиента |
| `PUT` | `/clients/{client_id}` | Обновить профиль |
| `GET` | `/clients/{client_id}/dates` | Список дат с данными |
| `GET` | `/topvisor/regions` | Регионы Topvisor |
| `GET` | `/providers` | Список LLM-провайдеров |
| `POST` | `/labels/import` | Импорт эталонной разметки |

Авторизация: `Authorization: Bearer <WEBHOOK_SECRET>` (кроме `/health`)

## Деплой

```bash
# 1. Клонировать репозиторий
git clone <repo-url> && cd serp

# 2. Прогнать тесты
pip install -r requirements-dev.txt && pytest  # 224 теста

# 3. Создать .env из шаблона
cp .env.example .env

# 4. Положить credentials.json рядом с docker-compose.yml

# 5. Запустить контейнер
docker compose up -d

# 6. Проверить
curl http://localhost:8000/health
```

## verify.sh

Автоматизированная верификация после деплоя (6 проверок):

1. **Тесты** — pytest в контейнере
2. **Health endpoint** — curl /health
3. **Container status** — docker compose ps
4. **Error logs** — grep логов (исключая INFO-метрики)
5. **Database schema** — проверка таблиц/колонок
6. **Data integrity** — нет осиротевших записей

Запуск: `./verify.sh` (exit 0 при 6/6 passed)
