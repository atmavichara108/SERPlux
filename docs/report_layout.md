# Раскладка листа «Отчёт» — канон

> Эталон: лист «Эталон разметки» (Лист1) в Google Sheets.
> Дата фиксации: 2026-07-11.

## Вертикальная структура (для каждого поискового провайдера)

| Строка | Содержимое |
|--------|-----------|
| 1 | `Позиции {провайдер} на {ДД.ММ.ГГГГ}` — в колонке A. Одна на весь блок провайдера. |
| 2 | Пустой буфер. |
| 3 | Имя субъекта (крупный шрифт) — в колонке начала блока субъекта (pos-колонка). |
| 4 | Название гео (страна) — в pos-колонке каждого субъекта. |
| 5–14 | 10 позиций: номер позиции в pos-колонке (заливка sentiment), URL в url-колонке. |
| 15 | Пустой буфер. |
| 16+ | Повтор для следующего гео: строка гео → 10 позиций → буфер. |

После последнего гео — два пустых строки-разделителя перед следующим провайдером.

## Горизонтальная карта колонок

### Правило буферов (асимметричное)

- **Колонка A** — всегда пустая (левое поле).
- **Субъект 1**: колонка B = pos (номера позиций, заливка), колонка C = URL.
- **Буфер после первого субъекта**: 3 пустые колонки (D, E, F).
- **Субъект 2**: колонка G = pos, колонка H = URL.
- **Буфер перед каждым следующим субъектом (≥3)**: 1 пустая колонка.
- **Субъект N (N ≥ 2)**: 2 колонки (pos + URL), перед ним 1 буферная колонка.

### Таблица колонок (0-indexed)

| Субъект | pos-колонка | url-колонка | Буфер перед | Буфер после |
|---------|-------------|-------------|-------------|-------------|
| 1 | 1 (B) | 2 (C) | — | 3 колонки (3,4,5 = D,E,F) |
| 2 | 6 (G) | 7 (H) | 1 колонка (5→6, т.е. F уже часть буфера) | 1 колонка (8 = I) |
| 3 | 9 (J) | 10 (K) | 1 колонка (8 = I) | 1 колонка (11 = L) |
| 4 | 12 (M) | 13 (N) | 1 колонка (11 = L) | 1 колонка (14 = O) |
| N (≥2) | 6 + (N-2)*3 | 7 + (N-2)*3 | 1 колонка | 1 колонка |

Формула для субъекта N (N ≥ 2):
- `pos = 6 + (N - 2) * 3`
- `url = pos + 1`

Для N=1: `pos = 1, url = 2, cols = 3`.

### Пример: 4 субъекта

```
Колонка: A  B  C  D  E  F  G  H  I  J  K  L  M  N  O
         ·  S1 S1 ·  ·  ·  S2 S2 ·  S3 S3 ·  S4 S4 ·
         ·  p  u  ·  ·  ·  p  u  ·  p  u  ·  p  u  ·
```

Где p = pos-колонка (номер + заливка), u = url-колонка.

## Заливка sentiment

Заливка применяется **к ячейке номера позиции** (pos-колонка), НЕ к URL и НЕ ко всей строке.

| Метка | Цвет | RGB |
|-------|------|-----|
| positive | Зелёный | (0.85, 0.92, 0.83) |
| negative | Красный | (0.96, 0.80, 0.80) |
| neutral | Жёлтый | (1.0, 0.95, 0.80) |

## Динамическое количество субъектов

Количество субъектов определяется из профиля клиента (`clients.queries`).
Раскладка строится функцией `_build_subject_layout()` в `reporter.py` для любого N.
