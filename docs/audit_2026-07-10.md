## Критичные дефекты (исправлены)

### 1. Статус прогона живёт только в памяти

- **Статус:** подтверждён баг → исправлен.
- **Где:** `webhook.py` (глобальный `_last_run`), отсутствовала таблица `run_status`.
- **Описание:** `_last_run` терялся при рестарте контейнера; прогон мог висеть в `running` часами.
- **Исправление:**
  - `storage.py`: добавлена таблица `run_status` (singleton `id=1`) и функции `get_run_status()` / `update_run_status()`.
  - `webhook.py`: удалён `_last_run`; `trigger_run` и `_run_pipeline` атомарно обновляют статус в БД; `GET /status` читает из БД.
  - `migrate.py`: `run_status` создаётся при миграции; `_verify_schema` проверяет её наличие.
  - `verify.sh`: таблица `run_status` включена в проверку схемы.
- **Тесты:** `tests/test_webhook.py::TestRunStatusPersistence` (2 новых теста), обновлены `TestStatusExtendedFields`.
- **Строки:** `storage.py:115-140`, `storage.py:661-720`, `webhook.py:40-315`, `migrate.py:188-215`.

### 2. Экспорт в Google Sheets без таймаута

- **Статус:** подтверждён баг → исправлен.
- **Где:** `exporter.py::_get_sheet()`, `reporter.py::_get_spreadsheet()`.
- **Описание:** вызовы `gspread` шли без `timeout`; при зависании Google API прогон не завершался.
- **Исправление:** после `gspread.service_account()` устанавливается `gc.http_client.timeout = (10, 60)` (connect/read) в обоих модулях.
- **Тесты:** `tests/test_exporter.py` (новый файл, 1 тест).
- **Строки:** `exporter.py:38`, `reporter.py:50`.

### 3. Labeler не пишет neutral / отсутствие confidence='uncertain'

- **Статус:** частично не воспроизведён как отдельный баг → найден и исправлен связанный дефект.
- **Где:** `labeler.py::_label_group_auto()`.
- **Описание:**
  - neutral действительно пишется для пустых сниппетов, ошибок провайдера и мусорных ответов LLM.
  - Однако `confidence` всегда оставался `'high'`, в том числе для neutral fallback. По контракту `contracts.md` и семантике "маркера неуверенности" должен быть `'uncertain'`.
- **Исправление:** для пустого сниппета и ошибки провайдера `confidence` теперь `'uncertain'`.
- **Тесты:** обновлены `tests/test_labeler_modes.py::test_auto_mode_snippet_fallback_to_neutral_on_empty_snippet` и `..._on_provider_error`.
- **Строки:** `labeler.py:174-204`.

### 4. Рассинхрон контракта `label_mode` (дополнительный блокер)

- **Статус:** подтверждён баг → исправлен.
- **Где:** `storage.py`, `migrate.py`, `labels.label_mode CHECK`.
- **Описание:** `webhook.py` и `labeler.py` работают с режимами `"auto"` / `"deep"`, но `CHECK` constraint в таблице `labels` и `storage.LABEL_MODES` допускали только `"domains"/"snippets"/"full"`. Новые метки не сохранялись бы в БД.
- **Исправление:**
  - `storage.py`: `LABEL_MODES = {"auto", "deep", "domains", "snippets", "full"}`; `CHECK` constraint обновлён.
  - `migrate.py`: пересоздание таблицы `labels` при устаревшем `CHECK`; новая схема включает `auto`/`deep`.
  - `main.py`: дефолт `label_mode` изменён с `"domains"` на `"auto"`.
- **Тесты:** обновлены `tests/test_storage_schema.py`, `tests/test_main.py`, `tests/test_webhook.py`.
- **Строки:** `storage.py:16`, `storage.py:117`, `migrate.py:223`, `main.py:52`.

- **Статус:** частично ok.
- **Где:** `apps_script.gs::_request()`.
- **Проверено:**

- **Статус:** подтверждено.
- **Где:** `storage.py::upsert_domain_label()`, `webhook.py::import_domain_labels()`.
