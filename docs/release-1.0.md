## Что вошло в v1.0

### Сбор данных
- **Topvisor Snapshots API:** сбор ТОП выдачи (10/20/50/100) по связкам запрос × гео × ПС
- **Поисковые системы:** Google, Яндекс (yandex.ru), Яндекс.com
- **Гео:** Литва, Германия, Великобритания, Лондон, ОАЭ, Кипр (настраивается через regions_map)
- **Идемпотентность:** повторный прогон не дублирует данные (INSERT OR IGNORE)

### Разметка тональности
- **Двухрежимная система:**
  - `auto` (дефолт): справочник domain_labels → LLM по сниппету → neutral при ошибке
  - `deep`: заглушка для v2 (обработка neutral по контенту страницы)
- **LLM-провайдер:** DeepSeek v4 Flash Free через opencode.ai/zen
- **Кэш domain_labels:** пары (domain, query, geo) с приоритетом source (manual_l1 > snippet > page)
- **Версионирование меток:** каждая разметка — новая версия, история сохраняется

### Отчётность
- **Накопительный режим:** новый блок отчёта вставляется СВЕРХУ листа «Отчёт», старые версии сдвигаются вниз
- **Хранение версий:** максимум 10 версий, старые обрезаются автоматически
- **Раскладка:** динамическая, из профиля клиента (N субъектов → N×2 колонок + буферы)
- **Заливка sentiment:** на ячейках номеров позиций (зелёный/красный/жёлтый)
- **Канон раскладки:** `docs/report_layout.md`, `docs/CANON.md`

### Мультиклиентность
- **Профиль клиента в БД:** queries, regions_map, searchers, geos, project_id
- **CRUD API:** `/clients` (GET/POST/PUT), `/clients/{id}` (GET/PUT)
- **UI:** dropdown client_id на листе «Настройки», меню «Клиенты» в Apps Script
- **Изоляция:** данные каждого клиента хранятся отдельно, переключение без правки кода

### Эталон разметки
- **Парсер Лист1:** `parseList1ToEtalon()` в apps_script.gs — разовый, запускается вручную
- **Импорт в БД:** `importEtalonToDb()` — батчами по 100 записей через `/labels/import`
- **Контракт:** query=имя субъекта (lowercase), geo=страна, domain из URL, sentiment по цвету
- **Источник истины:** `docs/CANON.md`

### Webhook API
- **11 эндпоинтов:**
  - `GET /health` — health-check (без авторизации)
  - `GET /status` — статус последнего прогона
  - `POST /run` — запуск пайплайна (сбор/разметка/экспорт/отчёт)
  - `GET /clients`, `POST /clients`, `GET /clients/{id}`, `PUT /clients/{id}` — CRUD клиентов
  - `GET /clients/{id}/dates` — список дат с данными
  - `GET /topvisor/regions` — регионы Topvisor
  - `GET /providers` — список LLM-провайдеров (read-only)
  - `POST /labels/import` — импорт эталонной разметки
- **Авторизация:** Bearer-токен (WEBHOOK_SECRET)

### Инфраструктура
- **Docker:** Dockerfile + docker-compose.yml, dev-зависимости включены в образ
- **CI:** GitHub Actions (pytest + shellcheck на push/PR)
- **Скрипты:** `verify.sh` (6 проверок), `backup_db.sh` (бэкап + ротация), `deploy.sh` (git pull → build → up → migrate)
- **Миграции:** `migrate.py` — идемпотентные, с бэкапом перед изменениями

### Тестирование
- **224 теста:** unit + integration, полностью изолированы (temp_db + моки gspread)
- **Покрытие:** все модули (collector, config, domain_labels, exporter, imports, labeler, main, migrate, parse_label, regions_map, reporter, storage, webhook)
- **Изоляция:** ни один тест не читает/пишет боевую БД или боевую таблицу

## Известные ограничения v1.0

### Legacy label_mode
- **Проблема:** `storage.LABEL_MODES` и CHECK `labels.label_mode` допускают устаревшие режимы `domains`/`snippets`/`full`
- **Влияние:** риск «зомби-меток» при ручных вставках
- **Статус:** отложено (требует миграции данных)

### Deep-режим — заглушка
- **Проблема:** режим `deep` не реализован (возвращает sentiment без изменений)
- **Влияние:** neutral результаты не уточняются по контенту страницы
- **Статус:** запланировано для v2.0

### Парсер эталона — ручной запуск
- **Проблема:** `parseList1ToEtalon()` запускается вручную через Run в редакторе Apps Script
- **Влияние:** нет автоматизации обновления эталона
- **Статус:** отложено (разовая операция, не критично)

### Watchdog для зависших прогонов
- **Проблема:** нет фонового наблюдателя для пометки зависших прогонов как `stalled`/`error`
- **Влияние:** оператор видит "прогон выполняется" часами при зависании внешнего вызова
- **Статус:** отложено для v1.1
