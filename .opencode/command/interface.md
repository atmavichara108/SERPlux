# Реализация веб-интерфейса SERPlux

## Контекст
!`cat docs/progress.md`
!`cat docs/techdebt.md`

## Задача
Реализуй веб-интерфейс SERPlux по приоритетам:

### 1. Дашборд (главная страница /)
- Последний прогон: статус, дата начала/конца, кол-во строк
- Кнопка «Запустить сбор»
- Быстрые метрики: всего прогонов, последний успешный, стоимость

### 2. Форма запуска (/run)
- Параметры: client_id, depth, with_labels, label_mode, date
- Валидация на клиенте и сервере
- После запуска — redirect на страницу статуса

### 3. Статус прогона (/status)
- Polling каждые 5 секунд
- Визуальная индикация: idle / starting / running / ok / error
- При завершении — показать статистику

### 4. История (/history)
- Таблица всех прогонов с фильтрацией по дате, клиенту, поисковику
- Клик по строке → детали прогона

## Требования
- FastAPI + Jinja2 шаблоны (templates/)
- Tailwind CSS (CDN для начала)
- Vanilla JS для polling
- НЕ трогай core-модули
- Обнови docs/progress.md после завершения
